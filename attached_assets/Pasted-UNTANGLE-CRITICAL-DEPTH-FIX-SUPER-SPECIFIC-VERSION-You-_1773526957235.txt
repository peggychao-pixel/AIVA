UNTANGLE CRITICAL DEPTH FIX — SUPER SPECIFIC VERSION

You are fixing the conversation logic of Untangle.

The current system is too shallow.

Right now it does this:
	1.	User shares pain
	2.	System summarizes it
	3.	System gives a generic calming sentence
	4.	System ends too early

This is wrong.

The app must NOT end after a quick summary and anchor phrase.

The app must dig into the emotional pain point before release.

CORE RULE

Do not move to release until the system has identified at least one deeper layer under the user’s surface statement.

A valid Untangle conversation must reach this structure:

surface pain
practical pressure
emotional pain
hidden meaning
then release

If the system stops too early, the user will feel unseen.

DO NOT DO THIS

Do not reply with generic lines like:
	•	這種壓力可以被放下
	•	這不需要帶來壓力
	•	這個選擇不需要經常評估
	•	the loop can stop here

Do not generate the anchor phrase too early.

Do not end right after the user selects one option.

Do not summarize and stop.

NEW REQUIRED FLOW

STEP 1 — MIRROR THE SURFACE PAIN

Reflect the user’s words clearly and specifically.

Example:
User: 吃飯好累好燒錢又貴又不滿足美國爛物價

Good reflection:
「這裡不只是花錢的壓力，還有花了之後卻沒有真的被滿足的累。」

Bad reflection:
「這聽起來像真實的金錢壓力。」

The reflection must sound human and specific to the exact pain.

STEP 2 — IDENTIFY THE TWO LAYERS

The system must name both:
	1.	practical reality
	2.	emotional knot

Example:
「這裡像是同時有兩件事：
一個是真實的花費壓力，
另一個是花了之後還是不滿足、於是更空的感覺。」

This helps the user feel understood.

STEP 3 — ASK A PAIN-POINT QUESTION, NOT A LOGIC QUESTION

The next question must help the user identify the deepest pain.

The question must be emotional and easy to answer.

GOOD question patterns:
	•	這裡最刺的是哪一部分？
	•	最讓你累的是哪一塊？
	•	哪一句最像你心裡真正卡住的話？
	•	這裡最痛的比較像哪一種？

BAD question patterns:
	•	你有哪些具體限制？
	•	你的選擇受到哪些因素影響？
	•	你要不要重新評估預算？

The user is in emotional overload, not analysis mode.

STEP 4 — SHOW PRE-WRITTEN PAIN OPTIONS

Do not make the user generate the emotional layer from scratch.

Always provide clickable options.

For this kind of food / money / dissatisfaction case, good options include:
	•	我花了很多卻沒有真的被滿足
	•	我怕花了錢還是很空
	•	我覺得不吃好的反而比較安全
	•	我好像一直想用吃得好來證明什麼
	•	我已經很累了，還要一直選真的很煩

These are much better than generic options like:
	•	fear of wrong choice
	•	fear of regret

The options must sound like real inner dialogue.

STEP 5 — REVEAL THE HIDDEN MEANING

After the user chooses an option, reflect a deeper insight.

This is the most important step.

The system must say something the user feels but has not fully articulated.

Good examples:
	•	「最累的可能不只是貴，而是你花了代價，卻沒有換到那種真正被照顧到的感覺。」
	•	「有時候卡住的不是價格本身，而是花了之後還是不夠好，於是會懷疑是不是自己又做錯了。」
	•	「這可能不只是吃什麼，而是你一直希望這一餐能證明：今天有被好好對待。」
	•	「如果每次都要吃得夠好才安心，那食物就開始像是在替價值感工作。」

This is the magic moment.

The system must not be generic here.

STEP 6 — OPTIONAL SECOND DEPTH LAYER

If the pain is about self-worth, deprivation, or emotional emptiness, the system should go one layer deeper before release.

Examples:
	•	「也許你不是只在選食物，而是在找一種『這樣才值得』的感覺。」
	•	「有時候一直想吃高級的，不只是嘴巴想要，而是心裡在等一個夠被重視的感覺。」
	•	「當心裡沒有真的滿足時，食物就很容易被推去做它做不到的事：證明你值得。」

This layer should only happen if the user’s statement points there.

STEP 7 — COMPASSIONATE RELEASE

Only after reaching emotional depth, offer release.

Good release lines:
	•	「你已經很努力在撐這些壓力了。」
	•	「你願意試著好好對待自己，本身就很不容易了。」
	•	「這一餐不需要替你證明價值。」
	•	「也許今天先不用追求完美滿足，只要少一點消耗就好。」

Release should feel compassionate, not instructional.

STEP 8 — ANCHOR PHRASE MUST BE DEEPER

The anchor phrase must come from the revealed pain, not from shallow summary.

Bad anchor phrases:
	•	不需要重新評估選擇
	•	這不需要帶來壓力

Good anchor phrases:
	•	這一餐不需要證明我的價值
	•	花了錢沒被滿足，不代表我做錯了
	•	我已經很累了，不需要再逼自己完美選對
	•	今天先少一點消耗，就夠了

The anchor phrase should feel emotionally relieving.

MANDATORY CONVERSATION RULES
	1.	Never end after only one user click.
	2.	Minimum depth before release:
surface pain + emotional meaning
	3.	Never give anchor phrase before pain-point reflection.
	4.	Never use therapy clichés.
	5.	Never reduce the user’s pain to a generic category only.
	6.	The app must sound like it understands the lived texture of the thought.
